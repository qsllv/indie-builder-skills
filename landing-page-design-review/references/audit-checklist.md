# Landing Page Audit Checklist

Use this rubric to evaluate whether a landing page communicates clearly, makes value perceptible, and supports conversion.

## Severity Levels

- P0: Blocks understanding, trust, or action. Fix before launch.
- P1: Weakens persuasion or conversion. Fix soon.
- P2: Improves polish, clarity, or flow. Fix after core issues.

## Core Checks

### 1. Positioning clarity

Pass if the first viewport explains:

- Product category or recognizable alternative.
- Target user or use case.
- Core problem.
- Concrete outcome.
- Primary CTA.

Flag as P0 when:

- The hero could describe many unrelated products.
- The page leads with company mission, technology, or slogans before user relevance.
- Visitors cannot tell whether the product is software, service, template, agency, community, or content.

### 2. Customer relevance

Pass if the page speaks to a specific user type, role, company stage, workflow, or situation.

Flag as P1 when:

- Copy addresses everyone.
- Benefits are not tied to a real workflow.
- The page uses internal company language instead of customer language.

### 3. Problem specificity

Pass if the page names the pain, cost, frequency, and context of the current workaround.

Flag as P1 when:

- The pain is generic: "save time", "work smarter", "be more productive".
- The page jumps to features before showing it understands the user's current problem.

### 4. Value perceptibility

Pass if users can imagine what improves after adoption.

Check each feature with:

Feature -> behavior change -> outcome -> emotional relief.

Flag as P0 when:

- The page lists features but never explains what the user gains.
- The value claim cannot be observed, measured, or imagined.

Flag as P1 when:

- Benefits are real but too abstract.
- Outcomes are buried below technical detail.

### 5. Information hierarchy

Pass if the page has a clear persuasion sequence:

Orient -> pain -> solution -> value -> proof -> action.

Flag as P1 when:

- Sections feel interchangeable.
- Multiple unrelated ideas compete in the same section.
- Important proof or CTA appears too late for the page's traffic intent.

### 6. Proof strength

Pass if the proof matches the risk of the action.

Low-friction actions can use product screenshots, demo previews, and example outputs.
High-friction actions need testimonials, case studies, customer logos, security detail, guarantees, or metrics.

Flag as P0 when:

- The page asks for payment, sales contact, migration, or sensitive data with little or no proof.

Flag as P1 when:

- Testimonials are vague.
- Metrics lack context.
- Screenshots are too small, fake-looking, or decorative.

### 7. CTA clarity

Pass if CTA text states the next valuable action.

Flag as P1 when:

- CTA text is vague: "Learn More", "Submit", "Get Started".
- Primary and secondary CTAs compete equally.
- The page has too many unrelated actions.

Flag as P0 when:

- There is no clear next step.
- CTA destination does not match the promise.

### 8. Visual usefulness

Pass if visuals answer "what happens when I use this?"

Strong visuals include:

- Product UI screenshots.
- Result previews.
- Before/after workflow states.
- Annotated examples.
- Short demo clips.
- Customer artifacts created by the product.

Flag as P1 when:

- Visuals are generic illustrations, abstract shapes, or mood images.
- Product screenshots are hidden, cropped beyond usefulness, or unreadable.
- Visual emphasis goes to decoration instead of comprehension.

### 9. Differentiation

Pass if the page explains why this option beats the user's current alternative.

Alternatives may include:

- Manual work.
- Spreadsheets.
- Notion or Airtable.
- Hiring an agency.
- Building in-house.
- A direct competitor.
- Doing nothing.

Flag as P1 when:

- The product sounds interchangeable.
- The page never mentions the old workflow or substitute.

### 10. Objection handling

Pass if the page anticipates major adoption worries:

- Price.
- Setup effort.
- Migration.
- Security.
- Data accuracy.
- Integration limits.
- Learning curve.
- Team adoption.
- Support.
- Refunds or cancellation.

Flag as P1 when:

- The page asks for commitment without answering obvious doubts.

### 11. Conversion path

Pass if the page has one primary action and repeat CTAs at natural decision points.

Flag as P1 when:

- CTA placement is sparse.
- CTA appears before value is established and is not repeated after proof.
- Secondary actions distract from the main conversion path.

## Empty Marketing Language

Treat these words as suspicious unless backed by a concrete example:

- Streamline
- Optimize
- Empower
- Transform
- Innovative
- Intelligent
- Seamless
- All-in-one
- Next-generation
- Robust
- Scalable
- Effortless
- Productivity platform

Rewrite empty claims into specific scenes, outcomes, or evidence.

## Common Failure Patterns

- Beautiful but unclear: strong visuals, weak positioning.
- Feature wall: many capabilities, no customer outcome.
- Abstract AI page: "AI-powered" everywhere, no concrete job.
- Proof gap: big promise, little evidence.
- CTA fog: visitors do not know what happens after clicking.
- Founder-first story: page talks about the company before user pain.
- Dashboard wallpaper: screenshots shown as decoration, not explained.
- Section pileup: many cards and claims without a persuasive order.
