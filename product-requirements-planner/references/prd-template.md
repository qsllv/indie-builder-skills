# PRD Template

Use this template for MVP or feature requirements.

- Product idea:

- Why now:

## 2. Problem

State the problem as a user situation, not a feature request.

Good:
"Indie makers collect feedback from email, Discord, and support chat, but cannot quickly see which requests are repeated."

Weak:
"Build an AI feedback dashboard."

## 3. Core Promise

"Help [target user] [achieve outcome] when [situation] without [pain]."

## 4. MVP User Flow

1. User arrives.
2. User provides input or connects source.
3. Product processes or organizes the input.
4. User sees a useful result.
5. User takes one next action.

## 5. Requirements

### Must Have

Only include capabilities required for first value, safety, trust, or basic operation.

### Should Have

Helpful, but not required for the first release.

### Later

Useful after validation.

### Not Now

Explicitly excluded to prevent scope creep.
